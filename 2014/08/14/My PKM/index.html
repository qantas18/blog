知识管理流程 (PKM)

总结和持续更新自己的个人知识管理(PKM)流程。

知识采集和管理

采集

知识的采集是一个碎片化的工作可能的场景如下：

  • 听别人说到某个知识点或者术语
  • 看书或者文章的时候看到推荐阅读书目
  • 看博客时看到推荐的链接
  • 突然想到的一个想法需要延生或者支撑
  • 定期阅读douban
  • 其他有待添加。。。

对于以上的场景，为了不打断正常的流程，采集的工具如下：

  • 不在电脑边：用手机端APP Ever Memo 记录
  • 在电脑边：
    • 如果是看到某个好的网站或者博客：用 Google 收藏夹
    • 如果是一篇文章，暂时没时间看：用 印象笔记 抓取
    • 如果是一个碎片或者有待查证：用 org-mode Capture 暂时记录

管理

以上收集的一些资料或者碎片信息，如果不定期的整理，那么除了让自己陷入一片混沌之中
外，起不到任何作用，所以收集只是最简单的一步，关键在于坚持定期管理或者说整理。

  • 用Org-Mode来整理Inbox中的碎片知识并排定日程查阅, 手机端app EverMemo的新添加内容会自动同步到印象笔记的EverMemo笔记本中，以同样的方式来整理
  • 用印象笔记整理Inbox中的内容，归类且加上标签，加入Org-Mode的阅读列表
  • 用Google收藏夹定期清理未分类的网址
  • 用Calibre来管理电子书 （Except for SAP PA, Managed with Folder）
  • 用Douban来管理书单？
  • 用Total Command + 百度云来管理文件类信息，包括以上的应用产生的文件

用Catelog和TAG来管理文件和内容。
Category一般分为这么几类，可用于ORG-MODE和印象笔记

  • 00INBOX: 临时的碎片信息，抓取的有待仔细阅读的网页
  • 00WKSPC: 本周的工作项所产生的一些输出，每一项单独用一个文件(ORG-MODE)或者一个笔记(印象笔记)来管理
  • 01IWORK: 当前所在的项目或者主要工作任务的信息
  • 02CLLCT: 收录的文章
  • 03DAIRY: 每天的晨写、每日备忘录、每天的当日总结、每周总结、每月总结、半年总结、年总结产生的输出
  • 04RECRD: 自己整理后产生的输出，一般可作为博客发表
  • 05INFOR: 常用的一些信息，比如一些资料，网址 等等
  • 06PAINT: 收录自己学习绘画的输出（先已经没时间练习了）
  • 99RULES: 一些给自己制定的规则，元数据信息，包括以上每个分类要放什么信息。

TAG可用来管理Calibre的Book，ORG-MODE和印象笔记
如何管理TAG，用XMID吧，树状结构

知识消费和提炼

知识吸收

收集的资料就是为了去看去消费的，对我自己来说，比较喜欢的一个输入来源就是电子书，其次是
网络上的资源。

  • 电子书
    • 电脑端：Calibre, 文件目录
    • 非电脑端：Kindle、iPad、手机
  • 收集的文章
    • 印象笔记
  • 网络资源（论坛，博客、个人主页）
    • Chrome / IE

知识提炼

知识提炼包括看书的笔记或者一些新的，或者是画出的一些脉络图。

  • 电脑端：Org-Mode
    • 读书笔记
    • 读书心得片段
    • 晨写
    • 周期性总结
    • 自己的一些工作流（GTD / PKM）
  • 非电脑端：纸质笔记 => 事后拍下来存入印象笔记中
  • 思维导图：XMIND

知识消化和创造

知识消化

定期Review之前提炼的一些内容和重新阅读原始知识来消化，并尝试根据自己的理解重画导图
或者流程图

  • 用Org-Mode做定期复习计划
  • 用XMIND来重画自己理解的概念
  • 用Powerpoint来画自己理解的流程图

知识创造

将消化的知识按照自己的理解重新generate出来就是一个创造的过程

  • 用Org-Mode写博文

知识分享和讨论

知识分享

将自己总结的知识分享出来，不但有益于他人共同提高，还可以结交到志同道合的朋友，
同时增加自己的知名度。

  • 个人博客
  • 个人维基

知识讨论

知识在讨论的过程中得到进一步理解加深和扩展。

  • 博客或者个人维基中积极回复网友的提问
  • 圈子或者群等社交媒体
